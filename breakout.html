自由分组话题讨论

TPAC 2023 期间，9月13日（周三）全天为全球与会者自由分组话题讨论，该环节将汇集 TPAC 全体与会者，以自由分组讨论的形式围绕当下数十个技术热点话题及标准化需求与趋势展开广泛交流。任何会员及公众均可自行提议希望得到广泛探讨的话题。分组话题安排将于9月11日发布在大会日程页面以及分组话题日历。欢迎自由提议分组话题！

提议新话题

如何提议一项新话题（展开参阅详细步骤）
  • 浏览已经提议了哪些话题（提议都在GitHub上）：欢迎通过表情符号、提问和评论的方式表达对某话题的支持。
  • 提议新话题：使用 GitHub 上的话题模版 创建新 issue，并提交如下内容：
    • Title（必填）：话题的标题（提示：自由分组话题旨在促进广泛交流，不得用于召开工作组/兴趣组/社区组会议）
    • Session description（必填）：提供话题简介，可使用简单的 markdown，可使用 inline formatting、links、lists，不要使用 headings。
    • Session goal（必填）：组织该话题的目的/预期目标
    • Additional session chairs（选填）：通常话题的提出者即被默认为该环节的主持人/组织者。若希望指定其他主持人/组织者，请在此提供相应信息。
    • <IRC channel（选填）：W3C 通常使用 IRC （https://irc.w3.org）作为在线聊天室，并在此记录和生成会议纪要，请指定你希望用于该话题的 IRC 频道名称（例如：#my-fav-session），与会者们可通过 IRC 频道进入话题讨论并获取会议记录；若无指定，W3C 将根据话题标题自动生成一个频道名称。
    • Who can attend（必填）：分组话题通常对公众开放（包括远程参会）；如有充分理由要求该话题仅面向 TPAC 的与会者开放，请在下拉列表选择“Restricted"。
    • Session duration（必填）：选择该话题所需时间（30或60分钟）
    • Other sessions where we should avoid scheduling conflicts（选填）：填写希望避免的时间段，例如想避免与其他某一（些）话题在同一时间段进行，请告知 W3C 相关话题编号。
    • Estimated number of in-person attendees（必填）：选择预期的现场参会人数（会务将据此安排合适大小的会议室）
    • Instructions for meeting planners（选填）：其他会务要求
    • Materials: Agenda, minutes, slides, etc.：可在此添加议程安排、参考资料等，话题结束后可在此添加讨论纪要。此部分仅支持添加链接。

浏览分组时间安排：每个时间段并行8个话题，如有需要可安排更多话题。

组织者可随时对所提议的话题进行修改（参见如何修改），包括会后添加会议记录等。但以下信息只有 W3C 团队方可修改：话题时间安排、会议室、所属的 track。

组织话题讨论

开始分组话题讨论

  • 提醒与会者知晓参会政策：《道德与职业行为准则》、《反垄断与竞争指南》
  • 介绍背景信息：
    • 该分组话题的形式：演讲、讨论或其他
    • 该话题的目的/预期目标
    • 话题的背景信息，方便与会者知悉讨论内容
  • 确定会议记录人员（不要求是 W3C 团队成员）：
    • 记录员应该（但非必须）在此话题的 IRC 频道记录会议内容（参见如何记录会议、如何使用 IRC）
    • 记录与会者名单（与会者在 IRC 频道输入 present+ name_surname 将参会姓名添加到与会名单）

结束分组话题讨论

  • 对话题讨论进行总结：
    • 本次讨论的要点、达成的一致意见、存在的不一致意见；
    • 接下来的步骤
    • 未来计划的执行者（可以是参与该话题的与会人员、相关的 W3C 工作小组、新成立社区组、W3C 团队成员等）
  • （在 IRC 频道输入 rrsagent, make minutes）保存会议记录：若话题本身对公众开放，则会议记录也会被公开；如话题本身未对公众开放，则会议记录仅限 W3C 会员获取。

分组话题讨论结束之后

  • W3C 会务团队将由 IRC 生成的会议纪要链接添加到相应话题，若您未使用 IRC，请将自动生成的纪要链接替换为正确链接。
  • 若希望提供会议总结及接下来的计划（如在哪里进行后续讨论），请通过 github issue comments 提交评论。
  • 进一步参阅分组话题内容分享政策：
    • 话题组织者确保以文字形式记录会议要点；
    • 话题组织者确保生成会议记录。TPAC 会务团队将与话题组织者协调将会议记录链接到相应话题；
    • W3C 可能在 w3.org 上为会议记录创建副本，用于持久访问。

更多提示

如何使用 GitHub lables：

在创建了一个新话题之后，你可能看到该话题被添加了一些标签。这些标签仅供 TPAC 会议策划者使用，与会者无法参与设置。策划者遵循以下规则使用这些标签：

  • "Session" 标签意味着此条目为一个 TPAC 自由分组话题（而非普通的 Github issue）
  • 使用标签将话题分配到一或多个由 W3C 团队的会议策划者建立的 tracks
  • 我们通过工具来验证分组话题数据并检测时间安排冲突，所生成的错误和警告均通过相应标签来体现

如何更新已有话题：

创建新话题时须使用话题模版。后续的更新请通过评论该话题的方式提交，避免直接编辑 markdown（添加链接除外）。例如：

  • 添加关于话题的更多信息
  • 为议程添加链接

请留意：

  • 不要改变话题模版所设置的格式
  • 不要将 Zoom 会议信息添加到话题描述中。Zoom 信息由 TPAC 会务团队统一管理并将发布在分组话题日历中。

如何撤销话题：

分组话题日期之前（9月13日之前），话题的提出者可以通过关闭话题 issue 的方式撤销该话题（选填：说明撤销理由）。除非是撤销话题，否则任何时候都请不要关闭该话题 issue。

如何更好地组织分组讨论：

  • 知悉参与政策
  • 观看视频 Inclusive Facilitation Training
  • 为了鼓励广泛参与、降低语言沟通障碍，我们建议您：
    • 时常总结讨论内容、检查与会者是否理解、必要时予以澄清；
    • 实时记录讨论内容（例如，使用 IRC：https://irc.w3.org）或使用 Zoom 上的转录字幕；
    • 语速适当且发音清晰，避免方言或行话；
    • 邀请鼓励大家参与讨论
  • 关于会议记录，详见如何记录会议、如何使用 IRC
  • 远程参会：请勿公开分享 Zoom 会议信息，以避免干扰。
